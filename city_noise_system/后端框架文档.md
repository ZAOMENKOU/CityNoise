# 城市噪音系统后端框架文档

## 1. 系统架构概述

城市噪音系统采用典型的Spring Boot + MyBatis Plus架构，遵循三层架构设计：
- **控制层（Controller）**：处理HTTP请求，参数校验，返回响应
- **业务逻辑层（Service）**：实现核心业务逻辑
- **数据访问层（Mapper）**：处理数据库操作

系统采用JWT进行身份认证，支持多角色权限控制，包括居民、管理员和处理人员。

## 2. 目录结构及功能说明

### 2.1 controller/ - 控制器层

**作用**：处理HTTP请求，参数校验，权限控制，返回响应结果

**主要控制器**：

| 控制器名称 | 主要功能 | 访问路径 | 权限要求 |
|---------|---------|---------|--------|
| `AdminController` | 管理员专用接口，管理用户、分配任务、统计数据 | `/api/admin` | 管理员角色 |
| `AuthController` | 用户认证接口，登录、注册、获取用户信息 | `/api/auth` | 部分接口无需认证 |
| `ComplaintController` | 投诉相关接口，提交投诉、查询投诉、评价投诉 | `/api/complaint` | 居民角色（提交），所有角色（查询） |
| `MonitorController` | 监控数据接口，获取噪音监控数据 | `/api/monitor` | 无需认证 |
| `NotificationController` | 通知管理接口，获取通知、标记已读 | `/api/notification` | 所有角色 |
| `WorkerController` | 处理人员专用接口，处理任务、更新进度 | `/api/worker` | 处理人员角色 |

**核心功能**：
- RESTful API设计
- JWT认证与权限控制
- 参数校验与异常处理
- 响应结果统一封装

### 2.2 dto/ - 数据传输对象

**作用**：定义请求和响应的数据结构，用于前后端数据交互

**主要DTO**：

| DTO名称 | 用途 | 包含字段 |
|--------|-----|---------|
| `ComplaintCreateDTO` | 投诉创建请求 | 标题、描述、噪音类型、经纬度、行政区、详细地址 |
| `LoginDTO` | 登录请求 | 用户名、密码 |
| `RegisterDTO` | 注册请求 | 用户名、密码、真实姓名、电话 |
| `RatingSubmitDTO` | 评价提交请求 | 评分、评价内容 |
| `TaskUpdateDTO` | 任务更新请求 | 状态、备注 |
| `UnsolvedSubmitDTO` | 标记未解决请求 | 未解决原因 |

**设计特点**：
- 专注于数据传输，不包含业务逻辑
- 字段验证和类型转换
- 前后端数据结构统一

### 2.3 entity/ - 实体类

**作用**：定义数据库表结构映射，对应数据库中的表

**主要实体**：

| 实体名称 | 对应表名 | 主要字段 |
|--------|---------|---------|
| `Complaint` | `complaints` | 投诉ID、用户ID、标题、描述、噪音类型、状态、创建时间、更新时间、删除标记 |
| `ComplaintImage` | `complaint_images` | 图片ID、投诉ID、图片路径 |
| `ComplaintLocation` | `complaint_locations` | 位置ID、投诉ID、经度、纬度、行政区、详细地址 |
| `HandlingRecord` | `handling_records` | 记录ID、投诉ID、操作人ID、处理人ID、操作类型、备注、创建时间 |
| `MonitorData` | `monitor_data` | 数据ID、监测站ID、噪音值、创建时间 |
| `MonitorStation` | `monitor_stations` | 监测站ID、名称、经度、纬度、行政区、详细地址 |
| `Notification` | `notifications` | 通知ID、用户ID、标题、内容、关联投诉ID、已读状态、创建时间 |
| `Rating` | `ratings` | 评价ID、投诉ID、用户ID、评分、评价内容、创建时间 |
| `User` | `users` | 用户ID、用户名、密码、真实姓名、角色、电话、头像、创建时间、删除标记 |

**设计特点**：
- 使用MyBatis Plus注解配置
- 支持软删除（@TableLogic）
- 自动填充时间字段（@TableField(fill = FieldFill.INSERT/UPDATE)）

### 2.4 filter/ - 过滤器

**作用**：拦截HTTP请求，进行认证和授权处理

**主要过滤器**：

| 过滤器名称 | 作用 | 处理逻辑 |
|---------|-----|---------|
| `JwtAuthenticationFilter` | JWT认证过滤器 | 验证Token有效性，解析用户信息，设置请求属性 |

**核心功能**：
- 白名单路径过滤
- JWT Token验证
- 用户信息解析与传递
- 认证失败处理

### 2.5 handler/ - 异常处理器

**作用**：统一处理系统异常，返回标准化的错误响应

**主要处理器**：

| 处理器名称 | 作用 | 处理范围 |
|---------|-----|---------|
| `GlobalExceptionHandler` | 全局异常处理器 | 处理系统各类异常，包括业务异常、参数异常等 |

**核心功能**：
- 异常捕获与分类处理
- 错误信息标准化
- 日志记录

### 2.6 mapper/ - 数据访问层

**作用**：定义数据库操作接口，使用MyBatis Plus实现CRUD操作

**主要Mapper**：

| Mapper名称 | 对应实体 | 主要方法 |
|---------|---------|---------|
| `ComplaintMapper` | `Complaint` | 投诉相关数据库操作 |
| `ComplaintImageMapper` | `ComplaintImage` | 投诉图片相关操作 |
| `ComplaintLocationMapper` | `ComplaintLocation` | 投诉位置相关操作 |
| `HandlingRecordMapper` | `HandlingRecord` | 处理记录相关操作 |
| `MonitorDataMapper` | `MonitorData` | 监控数据相关操作 |
| `MonitorStationMapper` | `MonitorStation` | 监测站相关操作 |
| `NotificationMapper` | `Notification` | 通知相关操作 |
| `RatingMapper` | `Rating` | 评价相关操作 |
| `UserMapper` | `User` | 用户相关操作 |

**设计特点**：
- 继承MyBatis Plus的BaseMapper
- 支持自定义SQL查询
- 批量操作支持

### 2.7 service/ - 业务逻辑层

**作用**：实现核心业务逻辑，协调各个组件完成业务功能

**主要Service**：

| `HandlingRecordService` | `HandlingRecordServiceImpl` | 处理记录管理 |
| `MonitorDataService` | `MonitorDataServiceImpl` | 监控数据管理 |
| `MonitorStationService` | `MonitorStationServiceImpl` | 监测站管理 |
| `NotificationService` | `NotificationServiceImpl` | 通知管理 |
| `RatingService` | `RatingServiceImpl` | 评价管理 |
| `UserService` | `UserServiceImpl` | 用户管理，包括登录、注册、信息更新等 |

**核心功能**：
- 业务逻辑实现

**作用**：提供通用工具方法，支持系统功能

**主要工具类**：

| 工具类名称 | 主要功能 | 方法说明 |

## 3. 核心业务流程

### 3.1 投诉处理流程

1. **投诉提交**：居民提交投诉，包含基本信息、位置信息和图片
2. **任务分配**：管理员将投诉分配给处理人员
3. **处理中**：处理人员开始处理投诉，更新处理进度
4. **处理完成**：处理人员标记投诉为已解决
5. **评价反馈**：居民对处理结果进行评价
6. **状态管理**：根据处理情况更新投诉状态

### 3.2 认证流程

1. **登录**：用户提交登录信息，系统验证并生成JWT Token
2. **Token验证**：请求经过JwtAuthenticationFilter，验证Token有效性
3. **权限检查**：控制器根据用户角色进行权限控制
4. **用户信息获取**：从Token中解析用户信息，用于业务处理

### 3.3 通知流程

1. **通知生成**：系统在关键节点（如任务分配、状态变更）生成通知
2. **通知发送**：向相关用户发送通知
3. **通知查看**：用户查看通知列表，标记已读

1. **完整的角色权限体系**：支持居民、处理人员、管理员三种角色
2. **响应式API设计**：RESTful风格，统一响应格式
3. **完善的异常处理**：全局异常捕获，标准化错误响应
4. **高效的数据访问**：MyBatis Plus优化数据库操作
5. **安全的认证机制**：JWT Token认证，密码MD5加密
6. **丰富的业务功能**：投诉管理、任务分配、评价系统、通知系统
7. **地图集成**：支持地理位置展示和附近投诉查询

## 6. 部署与维护

### 6.1 环境要求
- JDK 17+
- MySQL 8.0+
- Maven 3.6+

### 6.2 启动步骤
1. 配置数据库连接信息
2. 执行数据库初始化脚本
3. 运行 `mvn spring-boot:run` 启动应用
4. 访问 `http://localhost:8080/api/auth/login` 进行登录

### 6.3 监控与日志
- 系统使用SLF4J进行日志记录
- 关键操作均有日志输出
- 支持生产环境日志配置
