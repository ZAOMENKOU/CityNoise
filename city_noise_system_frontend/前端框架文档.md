# 城市噪音系统前端框架文档

## 1. 系统架构概述

城市噪音系统前端采用现代Vue 3技术栈，基于Vite构建工具，实现了完整的噪音投诉管理和监测功能。系统采用组件化设计，遵循MVVM架构模式，通过Vue Router实现路由管理，Pinia实现状态管理。

**核心技术栈**：
- Vue 3 + Composition API
- Element Plus UI库
- Vue Router 4
- Pinia 状态管理
- ECharts 数据可视化
- Vite 构建工具

## 2. 目录结构及功能说明

### 2.1 src/ - 源代码目录

**作用**：包含所有前端源代码，是项目的核心目录

### 2.1.1 assets/ - 静态资源

**作用**：存储静态资源文件，包括样式、图片等

**子目录**：
- `styles/` - 全局样式文件
  - `reset.css` - 样式重置文件
  - `variables.css` - CSS变量定义
  - `global.css` - 全局样式定义

**设计特点**：
- 使用CSS变量统一管理颜色、间距等样式属性
- 模块化样式管理，便于维护
- 响应式设计支持

### 2.1.2 components/ - 组件目录

**作用**：存放可复用的Vue组件

**子目录**：
- `business/` - 业务组件
  - `StatCard.vue` - 统计卡片组件
  - `StatusTag.vue` - 状态标签组件
- `common/` - 通用组件
  - `Button.vue` - 按钮组件
  - `ErrorOverlay.vue` - 错误提示组件
  - `Input.vue` - 输入框组件
  - `LoadingSpinner.vue` - 加载动画组件
- `layout/` - 布局组件
  - `Layout.vue` - 主布局组件（包含顶部导航和侧边栏）

**设计特点**：
- 组件化设计，提高代码复用性
- 业务组件与通用组件分离
- 布局组件统一管理页面结构

### 2.1.3 router/ - 路由配置

**作用**：配置应用的路由规则，实现页面导航

**主要文件**：

**核心功能**：
